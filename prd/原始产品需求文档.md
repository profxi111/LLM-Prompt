# 提示词智能调度与生成工具产品文档

---

## 一、产品概述

### 1.1 产品定位

本产品是一款**局域网专属的提示词智能生成与调度工具**，聚焦电商、海报设计等垂直场景，通过主Agent实现用户意图识别与专项子Agent智能分发，搭配类RAGFlow的提示词知识库检索与收藏功能，全程无需本地部署大模型、无需独立显卡，仅调用第三方厂商大模型API，为用户提供精准、个性化、可复用的垂直领域提示词生成服务，支持系统后续功能扩展与能力迭代壮大。

### 1.2 核心价值

1. **垂直场景精准适配**：针对电商、海报设计等细分领域打造专项提示词生成能力，告别通用提示词的泛化问题，贴合实际业务需求；

2. **智能调度降本提效**：主Agent自动识别用户需求与图片信息，无需人工选型，直接分发至对应专项子Agent处理，简化操作流程；

3. **个性化记忆与复用**：搭建专属提示词知识库，支持优质提示词收藏、语义检索，让生成内容持续贴合用户使用偏好，实现知识沉淀；

4. **多模型灵活兼容**：统一适配各类第三方厂商大模型API，无需本地运行模型，无独立显卡也可流畅使用，兼顾局域网访问与高阶生成需求；

5. **局域网安全可控**：支持局域网内部访问，核心数据存储与处理均在本地内网完成，仅调用外部模型API时联网，保障提示词、用户偏好等核心数据的安全性与私密性；

6. **低门槛易部署**：无需独立显卡、无需复杂AI环境配置，普通办公电脑即可搭建运行，降低使用门槛。

### 1.3 目标使用场景

适用于个人或小型团队在局域网环境下，开展电商产品宣传、海报设计、营销文案创作等工作，核心场景包括：

- 电商从业者快速生成产品标题、卖点、详情页提示词；

- 设计人员根据上传的参考图片，生成海报构图、风格、色彩相关提示词；

- 团队内部沉淀优质提示词，实现知识共享与复用；

- 低配置办公设备用户，无需独立显卡，通过外部API实现高质量提示词生成。

## 二、核心需求梳理

### 2.1 核心功能需求

1. **Agent智能调度体系**：搭建主Agent作为中枢，负责接收用户上传的图片+文字需求，识别用户意图（电商/海报/其他），并精准分发至对应专项子Agent；子Agent各司其职，电商Agent专注电商类提示词生成，海报Agent专注海报设计类提示词生成，后续可灵活扩展更多专项Agent。

2. **RAG提示词知识库**：实现类RAGFlow的语义检索功能，支持用户收藏优质提示词，系统自动将提示词存入向量库与数据库，支持按需求语义匹配历史优质内容，为新提示词生成提供参考，提升生成质量。

3. **多厂商大模型API兼容管理**：支持配置、管理各类第三方厂商大模型API（如通义千问、DeepSeek、MiniMax等），主Agent可根据场景自动分配模型，或支持手动切换模型，统一调用逻辑，无需针对不同模型修改核心代码，全程无需本地部署、无任何GPU依赖。

4. **图片+文字双输入**：用户可上传参考图片，搭配文字描述需求，系统结合图片信息与文字意图完成提示词生成，更贴合实际设计/营销需求。

5. **局域网访问部署**：支持在局域网内部署，同一网络下的多设备可通过IP地址访问工具，核心本地功能可离线操作，仅调用外部API时需联网。

6. **数据沉淀与管理**：存储用户提示词收藏记录、生成历史、个人偏好设置、模型API配置信息，支持用户查看、管理历史数据。

### 2.2 扩展需求

1. 支持新增更多专项子Agent（如UI设计、文案创作Agent）；

2. 支持用户自定义提示词偏好设置，固定风格、关键词、语气等参数；

3. 支持提示词编辑、导出、分类管理；

4. 支持模型调用优先级、参数自定义配置；

5. 支持多用户权限管理（团队场景）；

6. 支持外部API调用配额、费用统计。

### 2.3 非功能需求

1. **局域网兼容性**：支持Windows、Linux服务器部署，同一局域网内电脑、手机等设备通过浏览器正常访问；

2. **易用性**：界面简洁，操作流程简单，无需用户了解AI、代码相关专业知识，一键上传、生成、收藏；

3. **扩展性**：功能模块解耦，新增子Agent、模型API、功能模块无需重构核心系统；

4. **稳定性**：外部API调用异常时可自动切换备用模型，提示词生成响应及时，本地RAG检索、数据存储功能稳定；

5. **数据安全性**：所有本地数据存储于部署设备，不对外传输，API密钥加密存储，保障数据隐私；

6. **低配置适配**：无独立显卡、4G内存低配设备可流畅运行，无GPU依赖。

## 三、整体产品架构

本产品采用**分层解耦架构**，各模块独立分工又协同联动，兼顾当前功能落地与后续扩展壮大，全程无本地大模型推理模块，适配无显卡、全外部API方案，架构分层如下：

### 3.1 五层核心架构

1. **前端交互层**：负责用户界面展示与操作交互，包含图片上传、需求输入、提示词展示、收藏检索、偏好设置、模型配置等功能页面，采用原生HTML+CSS+JS实现，轻量易部署，适配浏览器访问。

2. **主Agent调度层**：系统核心中枢，接收前端传输的图片+文字需求，完成用户意图识别，判断需求所属场景（电商/海报），并根据配置分发至对应专项子Agent，同时路由至指定第三方大模型API，统筹整个生成流程。

3. **专项Agent执行层**：包含多个垂直场景专项子Agent，每个子Agent拥有独立的提示词生成逻辑与模板，仅负责对应场景的提示词生成工作，当前核心为电商提示词Agent、海报提示词Agent，后续可无限扩展。

4. **数据服务层**：分为**关系型数据库**与**RAG向量库**两部分，二者协同工作，纯CPU运行、无显卡依赖：

    1. **关系型数据库（SQLite）**：存储结构化数据，包括用户偏好、提示词原文、收藏记录、模型API配置、生成历史、子Agent参数等；

    2. **RAG向量库（FAISS-CPU）**：存储优质提示词的语义向量，实现语义检索，匹配与用户需求相似的历史优质提示词，为子Agent生成提供参考。

5. **模型适配层**：搭建统一模型适配器，**仅兼容第三方厂商大模型REST API**，屏蔽不同厂商API的调用差异，主Agent与子Agent只需通过统一接口调用模型，无需本地部署、无任何GPU依赖。

### 3.2 架构优势

1. **解耦性强**：各层模块独立，修改某一模块不影响其他功能，新增子Agent、模型只需对接对应接口，无需改动核心调度逻辑；

2. **数据协同**：数据库负责结构化数据存储与管理，RAG向量库负责语义检索，分工明确，解决"精确存储"与"模糊匹配"的双重需求；

3. **扩展灵活**：支持横向扩展子Agent、模型API、功能模块，适配系统后续发展壮大；

4. **低门槛适配**：全程无本地大模型、无显卡依赖，纯CPU运行，普通办公电脑即可部署；

5. **本地化友好**：整体架构轻量化，无需复杂部署环境，核心本地功能离线可用，仅调用API时联网。

## 四、核心功能模块详解

### 4.1 前端交互模块

核心面向用户，提供全流程可视化操作，包含6个子功能：

1. **需求输入模块**：支持图片上传（单张）、文字需求编辑框，用户可上传参考图片并描述需求（如"生成美妆产品电商标题提示词""科技风海报设计提示词"）；

2. **提示词生成模块**：展示生成进度、最终提示词内容，支持一键复制、编辑、收藏生成结果；

3. **提示词收藏模块**：用户可将优质提示词标记收藏，自动同步至数据库与RAG向量库，支持手动添加分类标签；

4. **RAG检索模块**：提供搜索框，支持关键词+语义双重检索，输入需求即可匹配历史收藏/生成的优质提示词，展示相似结果；

5. **个人偏好设置**：用户可配置常用风格、关键词、语气等偏好参数，子Agent生成提示词时自动贴合该设置；

6. **模型配置模块**：支持查看、编辑第三方厂商模型API配置信息，包括API地址、API Key、调用优先级、所属场景等，密钥加密存储，无需代码操作。

### 4.2 主Agent调度模块

系统核心中枢，承担三大核心任务：

1. **意图识别**：解析用户上传的图片信息与文字需求，结合场景标签，精准判断需求属于电商、海报还是其他场景，识别准确率覆盖核心场景；

2. **子Agent分发**：根据意图识别结果，自动将需求分发至对应的专项子Agent，若识别为未知场景，返回通用提示或引导用户自定义分类；

3. **模型路由**：根据子Agent场景与预设配置，自动分配适配的第三方大模型API，支持手动切换模型，单模型调用失败时自动切换备用API。

### 4.3 专项子Agent模块

当前内置两大核心专项Agent，后续可扩展：

1. **电商提示词Agent**：专注电商营销场景，生成内容包括产品标题、卖点提炼、详情页描述、营销文案等提示词，贴合电商转化需求，支持突出产品优势、关键词埋入等逻辑；

2. **海报设计提示词Agent**：专注海报设计场景，结合用户上传的参考图片，生成构图、色彩、光影、风格、镜头、元素搭配等设计类提示词，适配平面设计、海报创作需求。

### 4.4 RAG提示词知识库模块

实现提示词的沉淀、检索与复用，纯CPU运行、无显卡依赖，核心功能：

1. **双向存储**：用户收藏提示词后，系统将提示词原文、分类、标签存入数据库，同时通过轻量嵌入模型将提示词转为语义向量存入FAISS-CPU向量库；

2. **语义检索**：用户发起生成需求时，主Agent自动触发RAG检索，匹配Top3相似优质提示词，推送给子Agent作为生成参考；

3. **知识库管理**：支持查看所有收藏提示词，支持删除、编辑、分类整理，向量库同步更新数据。

### 4.5 多模型API适配模块

实现第三方厂商模型的统一管理，无本地模型、无GPU依赖，核心功能：

1. **模型配置管理**：数据库内置模型配置表，支持添加、编辑、删除各类第三方厂商模型（通义千问、MiniMax、DeepSeek等）API信息，密钥加密存储；

2. **统一适配器**：封装不同厂商模型的调用接口，主Agent与子Agent只需传入需求与模型ID，适配器自动识别厂商类型并完成API调用，统一返回生成结果；

3. **异常容错**：某一API调用失败时，适配器自动切换至同场景备用模型，保障提示词生成流程不中断；

4. **API调用日志**：记录每次外部API调用时间、模型、消耗情况，方便后续统计。

### 4.6 数据管理模块

负责所有结构化数据的存储与管理，核心存储内容：

1. **用户偏好数据**：风格、关键词、常用场景等；

2. **提示词数据**：收藏提示词、生成历史、分类标签；

3. **模型配置数据**：各厂商模型API地址、加密密钥、调用优先级、所属场景；

4. **调度记录**：主Agent意图识别记录、子Agent执行记录、API调用记录。

## 五、核心业务流程

### 5.1 用户提示词生成全流程

1. 用户通过前端页面上传参考图片，输入文字需求，提交生成请求；

2. 前端将图片、文字需求传输至后端，主Agent接收请求；

3. 主Agent先触发RAG检索，从FAISS-CPU向量库中匹配与需求相似的优质提示词，获取参考内容；

4. 主Agent解析图片与文字，完成意图识别，判断需求场景（电商/海报）；

5. 主Agent根据场景分发需求至对应专项子Agent，并路由至预设第三方大模型API；

6. 专项子Agent结合RAG检索的参考提示词、用户偏好，调用指定API生成专属提示词；

7. 生成结果返回至前端，用户查看提示词，可选择复制、编辑或收藏；

8. 若用户收藏，系统将提示词同步存入数据库与FAISS-CPU向量库，完成知识沉淀。

### 5.2 RAG检索与提示词收藏流程

1. 用户对生成的提示词满意，点击收藏按钮；

2. 后端将提示词原文、分类、创建时间、用户偏好等信息存入SQLite数据库；

3. 系统调用轻量CPU嵌入模型，将提示词内容转换为语义向量，存入FAISS-CPU向量库；

4. 下次用户发起相似需求时，主Agent将用户需求转为向量，与向量库中数据做相似度匹配；

5. 检索出相似度最高的3条提示词，传递给子Agent，辅助生成更贴合用户喜好的内容。

### 5.3 第三方模型API调用流程

1. 主Agent确定子Agent与场景后，读取数据库中该场景对应的模型API配置；

2. 主Agent将需求与模型ID传输至统一模型适配器；

3. 适配器根据模型ID查询加密配置，调用对应厂商大模型API；

4. 适配器获取API返回的生成结果，传递至子Agent；

5. 若调用失败，适配器自动切换至同场景备用API，重新调用；

6. 适配器将最终结果返回至子Agent，完成提示词生成。

## 六、软硬件与网络环境要求

### 6.1 硬件硬性要求（无显卡、无独显完全适配）

本产品全程无需独立显卡、无任何GPU依赖，集成显卡/核显即可，仅需满足基础办公设备配置，分最低配置与流畅配置两档：

1. **最低配置（满足单人/小型团队基础使用）**

    - CPU：双核及以上（任意现代办公CPU，Intel i3及以上/AMD同级均可）

    - 内存：4GB及以上

    - 硬盘：SSD固态硬盘，剩余存储空间≥10GB

    - 显卡：无要求，集成显卡/核显即可

2. **流畅配置（多人同时访问、高频使用）**

    - CPU：四核及以上（Intel i5及以上/AMD同级均可）

    - 内存：8GB及以上

    - 硬盘：SSD固态硬盘，剩余存储空间≥20GB

    - 显卡：无要求，集成显卡/核显即可

### 6.2 软件硬性要求（纯CPU、无显卡依赖）

1. **操作系统**：Windows 10及以上、Windows 11、macOS、Linux（Ubuntu 20.04及以上）均可

2. **核心运行环境**：Python 3.10~3.11（稳定版，无需其他运行环境）

3. **必备依赖库（纯CPU版本，一键安装）**

    - fastapi、uvicorn：后端Web服务与局域网部署

    - faiss-cpu：RAG语义向量检索（CPU专属版本，无GPU依赖）

    - sentence-transformers：轻量嵌入模型（CPU运行，实现向量转换）

    - pillow：图片处理

    - requests：第三方API网络请求

    - cryptography：API密钥 AES 加密存储

    - requests：第三方API网络请求

4. **无需安装的软件**

    - 无任何显卡驱动（CUDA、cuDNN等均无需配置）

    - 无需本地大模型部署工具（Ollama等无需安装）

    - 无需独立数据库服务（SQLite为Python自带，单文件存储）

    - 无需Docker、虚拟机等容器化工具

### 6.3 网络硬性要求

1. **部署设备网络**：必须可访问外网（仅用于调用第三方大模型API，本地功能可离线）

2. **局域网要求**：部署设备与访问设备需在同一局域网（同一WiFi/内网），支持设备间互相访问

3. **网络权限**：无需公网IP、无需端口映射、无需域名备案，纯内网工具访问

4. **网络稳定性**：外网连接稳定即可，保障第三方API调用顺畅

### 6.4 其他必备条件

1. **第三方大模型API凭证**：需提前申请所用厂商大模型的API Key、Secret等凭证，用于系统配置调用

2. **局域网IP**：部署设备需获取内网IP地址（如`192.168.1.xxx`），用于其他设备访问工具页面

3. **数据存储**：部署设备需保留足够本地空间，用于存储图片、数据库文件、向量库文件

## 七、技术选型方案

结合产品无显卡、全外部API、局域网部署、低门槛的核心需求，最终技术选型如下：

### 7.1 前端

**原生HTML + CSS + JavaScript**：无需复杂框架与打包流程，轻量易部署，适配浏览器访问，满足图片上传、交互操作、页面展示等核心需求，后期可升级为Vue/React。

### 7.2 后端

**Python + FastAPI**：Python拥有完善的RAG、网络请求生态，开发效率高；FastAPI轻量快速，支持图片上传、接口调用，适配局域网部署，启动简单，纯CPU运行无显卡依赖。

### 7.3 数据层

1. **关系型数据库：SQLite**：零配置、单文件存储，Python自带无需额外部署，适合局域网本地使用，后期可平滑迁移至PostgreSQL；

2. **RAG向量库：FAISS-CPU**：开源CPU专属向量库，检索速度快，无需外网、无显卡依赖，适配RAG语义检索需求；

3. **嵌入模型：m3e-small**：轻量CPU嵌入模型，实现提示词向量化，无需联网下载、无GPU依赖，秒级响应。

### 7.4 模型与AI服务

**统一模型适配器（Python自定义封装）**：仅兼容第三方厂商大模型REST API，不涉及本地模型部署，屏蔽不同厂商API调用差异，实现统一调度。

### 7.5 部署与网络

1. **部署环境**：Windows/Linux/macOS单机部署，无需服务器集群；

2. **网络配置**：后端监听`0.0.0.0`，支持局域网内多设备通过IP+端口访问；

3. **文件存储**：用户上传图片存储于本地uploads文件夹，路径存入数据库关联对应提示词。

## 八、局域网部署方案

### 8.1 部署前提

满足上述第六章软硬件与网络环境要求，提前准备好第三方大模型API凭证，确认部署设备可正常联网、局域网内设备互通。

### 8.2 部署步骤（无显卡、纯CPU版）

1. **环境安装**：在部署设备安装Python 3.10~3.11，勾选添加至系统环境变量

2. **依赖安装**：打开命令行，执行一键安装命令，安装纯CPU版本依赖库

```
pip install fastapi uvicorn faiss-cpu sentence-transformers pillow requests cryptography
```

3. **项目初始化**：解压项目代码，运行初始化脚本，自动创建SQLite数据库、FAISS-CPU向量库，无需手动配置

4. **API配置**：启动项目后，通过浏览器访问本地地址，进入模型配置页面，填写第三方厂商模型API信息（加密存储）

5. **服务启动**：运行启动命令，后端服务监听`0.0.0.0:8000`（端口可自定义）

6. **局域网访问**：查询部署设备内网IP（如`192.168.1.100`），同一局域网内其他设备通过浏览器访问`http://192.168.1.100:8000`即可使用

### 8.3 访问说明

1. 仅同一局域网（同一WiFi/内网）下的设备可访问；

2. 本地RAG检索、数据存储、页面访问功能可离线使用，仅提示词生成时需联网调用第三方API；

3. 无需外网映射、无需域名，内网IP即可访问，安全可控。

## 十、附录：核心概念说明

1. **主Agent**：系统中枢，负责意图识别与子Agent、模型API调度；

2. **专项子Agent**：垂直场景专属执行单元，负责对应场景提示词生成；

3. **RAG检索**：语义检索技术，通过向量匹配找到相似提示词，区别于数据库的精确关键词搜索，本产品为纯CPU版本；

4. **数据库**：SQLite数据库，存储结构化数据，负责提示词、配置、偏好的精确存储与管理，无需额外部署；

5. **[0.0.0.0](0.0.0.0)**：后端监听地址，允许局域网内所有设备访问，区别于仅本机访问的127.0.0.1；

6. **FAISS-CPU**：面向CPU的向量检索库，无显卡依赖，实现本地提示词语义检索；

7. **Ollama（产品文档1提及，本版本不采用）**：本地大模型服务工具，一键部署本地大模型，API调用简单，兼容主流大模型，满足局域网离线使用。本产品V1.0版本暂不采用本地模型方案，优先保障无显卡低门槛体验；

8. **sentence-transformers**：轻量Python嵌入模型工具，配合FAISS-CPU实现语义向量转换，支持CPU运行，无需GPU依赖。
